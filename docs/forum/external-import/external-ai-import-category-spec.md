# 外部AI相談ログ取り込み カテゴリー分類仕様書

## 1. 目的

外部AI相談ログ取り込み機能は、利用者が自分の ChatGPT 等で相談した会話ログやメモを、AI知恵袋Forumに投稿しやすい投稿候補へ変換するための機能である。

この仕様では、外部AIに渡すプロンプト、カテゴリー分類ルール、JSON出力形式、投稿前レビューの扱いを定義する。

目的は次の通り。

- 利用者の ChatGPT 等の会話ログを、AI知恵袋の投稿候補へ変換する
- 雑多な会話ログから、投稿に使える問題・質問・回答・論点・反論・補足を抽出する
- 経済、AI、特許、恋愛、仕事など複数テーマが混ざったログを扱えるようにする
- 投稿前に必ず利用者が内容をレビューする
- AI知恵袋側ではなく、利用者側の外部AIを下ごしらえに使うことで、AI知恵袋側の OpenAI API コストを抑える

この仕様は、AI知恵袋側で新たなOpenAI分類処理を増やすためのものではない。外部AIが投稿候補を整理し、AI知恵袋側はJSON取り込み、候補レビュー、類似スレッド確認、投稿実行に集中する。

## 2. 基本方針

外部AI相談ログ取り込みでは、完全自動投稿を行わない。

外部AIの出力は、あくまで投稿候補である。投稿前に利用者が必ず確認し、必要に応じて編集し、投稿するかどうかを選ぶ。

基本方針は次の通り。

- 完全自動投稿はしない
- 外部AIの出力は投稿候補であり、投稿前に必ずユーザーが確認・編集する
- プライバシー除去を最優先する
- 個人情報、第三者情報、私的すぎる情報は投稿候補に含めない
- カテゴリーは「利用者が指定する方法」と「AIが自動分類する方法」の両方を残す
- 投稿候補ごとの主カテゴリーは1つにする
- 関連カテゴリーは複数可とする
- 小カテゴリー・タグはAIに付与させる
- 似た論点は統合し、投稿候補数を増やしすぎない
- 投稿候補は最大10件程度に抑える

## 3. 2つの抽出モード

外部AI相談ログ取り込みでは、次の2つの抽出モードを用意する。

### A. テーマを選んで抜き出す

利用者が、投稿候補にしたい大カテゴリーを1つまたは複数選ぶモード。

例:

- 経済・政策
- AI・技術
- 特許・発明

この場合、外部AI用プロンプトには、選択したカテゴリーに関係する内容だけを抽出するよう指示する。

プロンプト方針:

```txt
選択したカテゴリーに関係する内容だけを投稿候補にしてください。
選択していないカテゴリーの話題は、投稿候補に含めないでください。
```

向いているケース:

- 長い会話ログの中から、特定テーマだけを抜き出したい
- 経済の話だけ、AIの話だけ、特許の話だけを投稿候補にしたい
- 雑談や別テーマを除外したい

### B. AIに全部分類させる

利用者がカテゴリーを指定せず、外部AIに会話ログ全体を読ませるモード。

外部AIは、投稿候補ごとに最も近いカテゴリーを選び、必要に応じて関連カテゴリー、小カテゴリー、タグを付与する。

投稿不要な雑談、個人的内容、プライバシー情報は除外する。判断が難しいものは「投稿不要」として扱い、公開投稿候補には含めない。

向いているケース:

- 会話ログに複数テーマが混ざっている
- 利用者自身が分類を決めにくい
- どのテーマが投稿候補になるかを外部AIに整理させたい

## 4. 大カテゴリー

初期表示する大カテゴリーは7個前後に抑える。

候補:

```txt
経済・政策
AI・技術
特許・発明
恋愛・人間関係
仕事・経営
生活・健康
その他
```

仕様:

- 表に出すカテゴリーは増やしすぎない
- 初期カテゴリーは10個以上にしない
- 利用者が選ぶのは大カテゴリーのみ
- 細かい小カテゴリー・タグは外部AIが付ける
- 将来的にカテゴリーが増えても、UI上は折りたたみや検索で扱う

カテゴリーが多すぎると、利用者が選ぶ前に迷いやすくなる。初期実装では大カテゴリーを絞り、小カテゴリーとタグをAI側の補助情報として扱う。

## 5. 複数カテゴリー指定

複数カテゴリー指定は許可する。ただし、最大3カテゴリー程度までとする。

理由:

- 1つだけだと不便な場合がある
- 何でも選べると分類が曖昧になる
- 4つ以上選ぶなら「AIに全部分類させる」モードの方が自然

仕様:

```txt
複数選択OK
最大3カテゴリーまで
投稿候補ごとの主カテゴリーは1つ
関連カテゴリーは複数可
```

例:

- 選択カテゴリー: 経済・政策、AI・技術
- 投稿候補A: `main_category = "経済・政策"`, `related_categories = ["AI・技術"]`
- 投稿候補B: `main_category = "AI・技術"`, `related_categories = ["経済・政策"]`

## 6. 主カテゴリーと関連カテゴリー

投稿候補ごとに、次の2つの分類項目を持たせる。

```txt
main_category
related_categories
```

方針:

- `main_category` は必ず1つ
- `related_categories` は0個以上
- UI一覧やDB上の主分類は `main_category` を使う
- 関連導線・タグ表示には `related_categories` を使う

例:

```json
{
  "main_category": "経済・政策",
  "related_categories": ["AI・技術"],
  "title": "AIを使った経済政策議論の整理"
}
```

`main_category` は投稿候補の主な置き場所を決めるための分類である。`related_categories` は、別テーマとも関係する投稿候補を見つけやすくするための補助分類である。

## 7. 小カテゴリー・タグ

小カテゴリーとタグは、利用者に選ばせずAIに付与させる。

例:

```txt
大カテゴリー: 経済・政策
小カテゴリー: 消費税、財政政策、金融政策、雇用、社会保障
タグ: 需要不足、減税、アベノミクス、インフレ、デフレ
```

仕様:

- `sub_category` は任意
- `tags` は複数可
- タグは3〜8個程度
- 抽象的すぎるタグは避ける
- 個人名や私的情報をタグにしない

望ましいタグ:

- 論点を検索しやすい
- 具体的なテーマを表す
- 投稿候補の内容と直接関係する

避けるタグ:

- 「重要」「悩み」「相談」など抽象的すぎるもの
- 個人名、店名、勤務先など特定につながるもの
- 会話ログ内だけでしか意味が分からないもの

## 8. プライバシー除去ルール

外部AI用プロンプトには、投稿候補生成前に必ず匿名化・除去するよう指示する。

除去対象:

- 個人名
- 住所
- 電話番号
- メールアドレス
- 店名や勤務先など個人特定につながる情報
- LINE内容など相手のプライバシーに関わる情報
- 恋愛、家族、健康、金銭など投稿に不要な私的情報
- 第三者を特定できる表現

方針:

- 判断に迷う情報は投稿候補に含めない
- 必要なら一般化する
- 投稿候補は不特定多数に見られても問題ない形にする
- 投稿候補は、公開掲示板に出してもよい内容だけにする

一般化の例:

```txt
特定の会社名 → 勤務先
実名の友人 → 知人
具体的な住所 → ある地域
個別のLINE内容 → 個人的なやりとり
```

## 9. 外部AI用プロンプト仕様

外部AI用プロンプトは、利用者が自分の ChatGPT 等に貼り付けるためのものとする。AI知恵袋側でこのプロンプトをOpenAI APIへ送ることは、初期実装では行わない。

### カテゴリー指定モード用プロンプト

```txt
以下の会話ログを、AI知恵袋Forumに投稿しやすい投稿候補へ整理してください。

抽出対象カテゴリー:
{{selected_categories}}

ルール:
- 選択したカテゴリーに関係する内容だけを抽出してください。
- 選択していないカテゴリーの話題は投稿候補に含めないでください。
- 投稿候補は最大10件までにしてください。
- 似ている論点・重複する質問は、別々に出さず1つに統合してください。
- 完全な自動投稿ではなく、投稿前レビュー用の候補として作ってください。
- 個人情報、第三者情報、私的すぎる情報は必ず除去・匿名化してください。
- 判断に迷う情報は投稿候補に含めないでください。
- 各投稿候補に main_category を1つ付けてください。
- 必要に応じて related_categories、sub_category、tags を付けてください。
- tags は3〜8個程度にしてください。

出力形式:
- 必ずJSONだけを出力してください。
- JSON以外の説明文は出力しないでください。
- posts 配列に投稿候補を入れてください。

会話ログ:
{{conversation_log}}
```

### AI自動分類モード用プロンプト

```txt
以下の会話ログを、AI知恵袋Forumに投稿しやすい投稿候補へ整理してください。

カテゴリー候補:
- 経済・政策
- AI・技術
- 特許・発明
- 恋愛・人間関係
- 仕事・経営
- 生活・健康
- その他

ルール:
- 会話ログ全体を読み、投稿候補ごとに最も近いカテゴリーへ分類してください。
- 投稿不要な雑談、個人的内容、プライバシー情報は除外してください。
- 各投稿候補に main_category を必ず1つ付けてください。
- 必要に応じて related_categories を付けてください。
- sub_category と tags も付けてください。
- 投稿候補は最大10件までにしてください。
- 似ている論点・重複する質問は、別々に出さず1つに統合してください。
- 完全な自動投稿ではなく、投稿前レビュー用の候補として作ってください。
- 個人情報、第三者情報、私的すぎる情報は必ず除去・匿名化してください。
- 判断に迷う情報は投稿候補に含めないでください。

出力形式:
- 必ずJSONだけを出力してください。
- JSON以外の説明文は出力しないでください。
- posts 配列に投稿候補を入れてください。

{{conversation_log}}
```

## 10. JSON出力形式

外部AIの出力形式は、次の形式を基本とする。

```json
{
  "posts": [
    {
      "main_category": "経済・政策",
      "related_categories": ["AI・技術"],
      "sub_category": "財政政策",
      "tags": ["消費税", "需要不足", "減税"],
      "title": "消費税減税は需要不足対策になるのか",
      "question": "消費税減税は日本経済の需要不足を改善する手段になり得るのか。",
      "claim": "消費税減税は、家計の可処分所得を増やし、需要不足を緩和する可能性がある。",
      "reason": "消費税は消費時点に課税されるため、税率が高いほど消費を抑制しやすい。",
      "premise": "日本経済には需要不足が残っているという前提に立つ。",
      "counterpoint": "財源やインフレ再燃を懸念する反論がある。",
      "notes": "個人情報は除去済み。"
    }
  ]
}
```

必須項目:

- `main_category`
- `title`
- `question`
- `claim`

推奨項目:

- `related_categories`
- `sub_category`
- `tags`
- `reason`
- `premise`
- `counterpoint`
- `notes`

### 現行実装との関係

現行の `ExternalAiImportModal` は、外部AIのJSON配列を読み取り、投稿候補として表示する実装になっている。

カテゴリー分類仕様を実装する場合は、次のどちらかで互換性を持たせる。

- `posts` 配列を読み取り、内部の投稿候補配列へ正規化する
- 既存のJSON配列形式も受け付け、`main_category` などがあれば追加情報として扱う

初期実装では、既存の投稿候補レビュー体験を壊さず、カテゴリー項目を追加情報として表示する。

## 11. 投稿前レビュー

外部AIの出力は、読み取っただけでは投稿されない。

投稿前レビューの流れ:

1. 利用者が外部AI用プロンプトをコピーする
2. 自分の ChatGPT 等に貼り付ける
3. 整理したい会話ログやメモも貼る
4. 外部AIが返したJSONをコピーする
5. AI知恵袋側の外部AI取り込み画面に貼り付ける
6. AI知恵袋側でJSONを読み込む
7. 投稿候補一覧を表示する
8. 利用者が1件ずつ確認する
9. 投稿する / 投稿しない / 編集する を選ぶ
10. 必要に応じて類似スレッドを確認する
11. 既存スレッドに投稿するか、新規スレッドを作成するかを選ぶ

仕様:

- すぐに自動投稿しない
- 利用者が投稿候補を確認できる
- 利用者が投稿候補を編集できる
- 類似スレッド確認ができる
- 既存スレッドに投稿するか、新規投稿するかを選べる
- 投稿しない場合でも、将来的には非公開ログに保存できる余地を残す

## 12. 類似スレッド・重複投稿との関係

外部AIで作った投稿候補が、既存スレッドと似ている場合でも、自動削除・自動統合はしない。

方針:

- 似た既存スレッドがあっても自動削除しない
- 似た既存スレッドがあっても自動統合しない
- 「似た既存スレッドがあります」と提示する補助機能にする
- 最終判断は利用者に任せる
- 公開投稿にせず、非公開ログとして保存する将来案を残す
- 参考になる既存投稿を保存できる将来案を残す

この方針により、重複投稿を減らしつつ、利用者の判断を優先する。

## 13. AI知恵袋側の役割

外部AIが下ごしらえを行い、AI知恵袋側は投稿前レビューと投稿先選択に集中する。

AI知恵袋側の役割:

- JSON取り込み
- 投稿候補レビュー
- 類似スレッド確認
- 投稿先選択
- 投稿実行
- 非公開保存
- 関連スレッドとの接続
- 必要最小限の検証

初期実装では、AI知恵袋側で追加のOpenAI分類処理を増やさない方針とする。

## 14. 認証との関係

ベータ認証の方針は `docs/forum-beta-auth-spec.md` に従う。

認証との関係:

- 外部AI取り込みから投稿する場合、投稿・新規スレッド作成はログイン必須
- AI整理系APIもログイン必須
- 未ログイン時は外部AI取り込みから投稿実行しない
- 投稿候補の編集UIは閲覧できても、投稿実行時はログイン必須
- 今後、保存系APIもログイン必須化する予定

現時点でログイン必須の代表API:

```txt
POST /api/forum/add-post
POST /api/forum/create-thread-from-draft
POST /api/forum/generate-issue
POST /api/forum/organize-post
GET  /api/forum/thread-summary
```

今後ログイン必須化を検討するAPI:

```txt
POST /api/forum/save-private-log
GET  /api/forum/private-import-logs
POST /api/forum/search-related
```

## 17. 実装時の注意点

カテゴリー分類仕様を実装する場合は、次の点に注意する。

- 既存の投稿候補読み取り処理を壊さない
- 既存のJSON配列形式との互換性をできるだけ残す
- `main_category` がない場合は、既存の `category` または「その他」へフォールバックする
- `related_categories` は配列として扱う
- `tags` は配列として扱う
- 文字列以外のタグや空文字は除外する
- 投稿候補数は最大10件程度に制限する
- 投稿実行前にログイン状態を確認する
- AI知恵袋側で追加OpenAI分類を行わない
- 個人情報や私的情報を投稿候補に含めない注意文を維持する
